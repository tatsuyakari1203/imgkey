# 09 - ImgKey Geometric Benchmark and Default Tuning

Date: 2026-05-18
Status: In progress
Owner: ImgKey Classical Quality / Benchmarking
Scope: Build a comprehensive synthetic asset benchmark, tune no-AI classical keyer defaults, and verify CPU/GPU parity.

---

## 1) Goal

Create a deterministic benchmark image suite that simulates real asset-keying cases, run ImgKey against multiple key colors, compare the result to known ground truth, and update defaults to the best robust settings.

The benchmark must include:
- transparency bands/ramps,
- small dots/speckles,
- 1px/2px/thin straight lines,
- diagonal lines and curves,
- hard edges and anti-aliased edges,
- holes/interior key-colored regions,
- black/white/red/colored foregrounds,
- multiple key backgrounds: current approved blue, green, cyan-ish, and uneven gradients.

The output must include detailed metrics, diagnostic images, and a clear recommended default profile.

---

#### P2.2 - Tune score weights and stop conditions
- Define score weights for asset-keying priorities:
  - thin line/detail recall high priority,
  - background leak high priority,
  - key residual high priority,
  - foreground core RGB preservation medium/high,
  - alpha MAE medium.
- Stop condition: do not accept a new default if it worsens background leak or foreground core RGB beyond tolerance even if detail improves.
- Objective default promotion rule:
  - global default may change only if weighted score improves by at least 5% over current default across all key colors,
  - thin-line/detail recall must not regress on any key color,
  - background leak and false foreground loss must stay within existing smoke tolerances or improve,
  - foreground core RGB delta must remain `<= 5` unless a stricter existing threshold applies,
  - green/cyan/gradient legacy fixtures must not regress.
- If the strict screenshot profile wins only blue/graphic cases or regresses green/cyan/gradient/legacy fixtures, add it as a named preset (for example `Asset Strict`) instead of making it global default.

Acceptance:
- `summary.json` clearly names best profile and reasons.
- Existing `python smoke_test.py` remains strict and passes.
- Geometry correctness is evaluated on CPU and, when compact CUDA DLL is available, benchmark-level CPU vs GPU parity must compare geometry RGBA/metrics. Existing GPU tile parity alone is not sufficient for v9.
